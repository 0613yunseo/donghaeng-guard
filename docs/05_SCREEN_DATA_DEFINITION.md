# 1. 문서 작성 기준

본 문서는 각 화면에서 필요한 데이터를 정의한다.

상세 API URL, Request/Response 구조, DB 컬럼명, PK/FK 관계는 이후 문서에서 별도 정의한다.

```
화면별 기능 정의서
→ 화면별 데이터 정의서
→ API Mapping
→ ERD 요구사항
→ API 명세서

```

# 2. 공통 데이터 기준

## 2.1 사용자 식별 데이터


| 데이터명       | 설명            | 출처    | 저장 여부 | 비고                  |
| ---------- | ------------- | ----- | ----- | ------------------- |
| userId     | 사용자 식별값       | 백엔드   | 저장    | MVP에서는 임시 userId 가능 |
| nickname   | 사용자 이름 또는 표시명 | 앱/백엔드 | 저장    | 선택값                 |

# 3. 사용자용 App 화면별 데이터 정의

## 3.1 온보딩 / 권한 안내 화면


| 항목     | 내용                   |
| ------ | -------------------- |
| 화면 목적  | 앱 사용 전 서비스 목적과 권한 안내 |
| 백엔드 연동 | 없음 또는 낮음             |
| 저장 필요  | 로컬 저장 중심             |



| 데이터명                   | 설명              | 출처  | 저장 위치 | 필수 여부 |
| ---------------------- | --------------- | --- | ----- | ----- |
| isFirstLaunch          | 최초 실행 여부        | 앱   | 로컬    | 필수    |
| bluetoothPermission    | Bluetooth 권한 상태 | OS  | 로컬    | 필수    |
| locationPermission     | 위치 권한 상태        | OS  | 로컬    | 필수    |
| notificationPermission | 알림 권한 상태        | OS  | 로컬    | 필수    |


---

## 3.2 간편 시작 / 로그인 화면


| 항목     | 내용           |
| ------ | ------------ |
| 화면 목적  | 사용자 식별 기준 생성 |
| 백엔드 연동 | 있음           |
| 저장 필요  | userId 저장 필요 |



| 데이터명        | 설명      | 출처    | 저장 위치 | 필수 여부 |
| ----------- | ------- | ----- | ----- | ----- |
| userId      | 사용자 식별값 | 백엔드   | 로컬/DB | 필수    |
| loginType   | 로그인 방식  | 앱     | 로컬/DB | 선택    |
| nickname    | 사용자 표시명 | 앱/백엔드 | DB    | 선택    |
| accessToken | 인증 토큰   | 백엔드   | 로컬    | 후순위   |
| userStatus  | 사용자 상태  | 백엔드   | DB    | 선택    |


MVP에서는 로그인 기능을 깊게 구현하지 않고, **간편 시작 + 임시 userId** 방식으로 처리 가능하다.
